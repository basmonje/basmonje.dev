JSDoc es una API que genera documentación a través de comentarios directamente en nuestro código. Esto nos sirve para saber que tipos de datos nos estan pidiendo, se podría decir que se cumple el mismo objetivo que con los `types` de `Typescript`.

Pero eso no es todo, `vscode` nos permite utilizar una función de comprobación de tipos e informe de errores de Typescript en archivos normales de Javascript, anteponiendo el siguiente comentario, en la primera línea de código:

```js
// @ts-check
```

Esta es una excelente manera de detectar errores comunes de programación.

Ejemplo:

```js
// @ts-check

/** @type {string} */
let foo;

/** @type {string | number} */
let b;

/** @type {number[]} */
let w = [1, 2];

/** @type {{ name: string, age: number }} */
const person = {
  name: "Javier",
  age: 15,
};
```

> Para leer mas sobre las [características avanzadas de Javascript](https://code.visualstudio.com/docs/nodejs/working-with-javascript) para vscode, si necesitas la [documentación de JSDoc](https://jsdoc.app/about-getting-started.html).
