## Programación Funcional

### ¿Por qué la programación funcional puede beneficiarnos?

Una razón sería escribir código facil de leer, como se muestra en los siguientes ejemplos:

Estructura imperativa

```javascript
for (let i = 0; i < products.length; i++) {
  products[i].price = Math.floor(product.price);
}
```

Estructura declarativa

```javascript
products.map((product) => {
  product.price = Math.floor(product.price);

  return product;
});
```

### Como disminuir lineas de código con métodos

**Array.prototype.find**
Se utiliza para buscar un elemento específico que supera la prueba, devuelve la primera coincidencia

```javascript
products.find((product) => product.onSale);
```

**Array.prototype.filter**
Se utiliza para buscar los elementos que pasan la prueba, devuelve cada coincidencia

```javascript
products.filter((product) => product.onSale);
```

**Array.prototype.every**
Si cada elemento cumplo los criterios, devolverá `true`

```javascript
products.every((product) => product.name);
```

**Array.prototype.some**
Si al menos un elemenot coincide con los criterios, devolverá `true`

```javascript
product.some((product) => product.onSale);
```

**Array.prototype.map**
Utilizado para transformar una matriz, devuelve una nueva

```javascript
products.map(product => {
    product.price => Math.floor(product.price);

    return product;
})
```

**Array.prototype.reduce**
Se utiliza para generar un único valor a partir de una matriz

```javascript
products.reduce((accumulated, product) => accumulated + product.price, 0);
```

### Encadenamiento de funciones

```javascript
const round = (num) => Math.floor(num);
const isDivisibleByTwo = (num) => num % 2 === 0;
const add = (accumulated, num) => accumulated + num;

const numbers = [0, 1.2, 2.4, 3.6, 4.8, 5, 6.2, 7.4, 8.6, 9.8];

const sum = numbers.map(round).filter(isDivisibleByTwo).reduce(add, 0);
```

### Currying

Currying es una técnica en la que se llama a una secuencia de funciones con un argumento en lugar de llagar a una función con varios argumentos. Cada función devuelve otra función. La función al final de la cadena devuelve el valor esperado real.

```javascript
// Instead of
const add = (a, b, c) => a + b + c;

add(2, 2, 2);

// Currying does
const curry = (a) => {
  return (b) => {
    return (c) => {
      return a + b + c;
    };
  };
};

curry(2)(2)(2);
```

### Recursión

La recursividad se produce cuando una función sigue llamándose a símisa hasta que se cumple alguna condición. En el ejemplo siguiente, estamos contando hacia abajo de 100.


### Inmutabilidad

Cuando hablamos de variables y objetos inmutables, simplemente queremos decir que una vez declaramos, su valir no se puede cambiar. Esto puede reducir la complejidad del codigo y hacer que la implementación sea menos propense a errores.

```javascript
const presents = ["🎁", "📦", "🎀", "💝", "🎄"];

// Mutable solution
presents.shift();

// Inmutable solution
const newPresents = presents.slice(1);
```
