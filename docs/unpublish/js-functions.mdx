Para ejecutar una función, se escribe su nombre y se le añade un juego de paréntesis.

Una función puede ser invocada múltiples veces, según sea necesario.

```javascript
function calculateAgeSegment() {
  const myAge = 26;

  if (myAge > 65) {
    console.log("Usted puede jubilarse");
  } else if (myAge > 18) {
    console.log("Usted es mayor de edad");
  } else {
    console.log("Usted es menor de edad");
  }
}

calculateAgeSegment();
```

## Argumentos

Una función puede recibir valores que son necesarios para su funcionamiento, puede recibir tanto valores como variables.

Esto expande el uso de las funciones como un procesamiento de código agnóstico de los valores que se necesitan procesar.

```javascript
function calculateAgeSegment(age, retirement) {
  if (age > retirement) {
    console.log("Usted puede jubilarse");
  } else if (age > 18) {
    console.log("Usted es mayor de edad");
  } else {
    console.log("Usted es menor de edad");
  }
}

calculateAgeSegment(26, 65);
```

> Es importante tener en cuenta el orden de los argumentos que una función recibe, ya que estos deben ser declarados en la invocación de la función.

## Retornar un valor

Podemos esperar que la función nos entregue un resultado al código que esta ejecuta, lo que nos permitirá guardar ese valor en una variable que estará disponible para su uso en el resto de nuestro código.

Para esto debemos declarar con return que esperamos que una función devuelva o responda.

```javascript
function suma(num1, num2) {
  return num1 + num2;
}

const result = suma(2, 2);

console.log(result); // 4
```

> El uso de return corta la ejecución del código dentro de la función.
> El código que se añade posterior a un return, nunca es ejecutado.
> Un return que no posee un valor nos devuelve undefined

# Hoisting

Siempre que se ejecuta un código en Javascript sucede el `hoisting` o `“levantamiento de variables”`, con esto las declaraciones de variables y funciones se elevan a la parte superior de nuestro código, y por eso Javascript nos permitiría usarlas antes de incluso declararlas.

Pero, en el caso de las variables, eleva la declaración de la variable y no su asignación, por lo tanto, si la usamos antes que nosotros le asignemos un valor, se encontrará `undefined`.

```javascript
console.log(num);

add();

var num = 8;

function add() {
  console.log("He invocado a add");
}

// undefined
// He invocado a add
```

> Este comportamiento genera comportamientos de código inesperados o que son difíciles de mantener seguros.
>
> Para evitar esto, ECMAScript nos entrega la opción de usar const y let, ya que el hoisting levantará todas las variables declaradas con var, pero estas otras detectará que se están >“usando” antes de su propia declaración o asignación.
